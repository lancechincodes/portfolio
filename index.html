RPG Battle Game

Inspiration:

I was inspired by the retro video game Samurai Shodown to build a RPG battle game where two characters challenge each other in a duel. But instead of choosing from a bank of predefined character abilities, my console game allows users to set unique health levels, mana, and attacks for either a wizard, warrior, or archer.

What it does:

This program includes subclasses of specific characters that derive from a base abstract class – each with a unique range of attributes, abilities, and interactions. It also demonstates object oriented programming, inheritance, and polymorphism to fulfill the objective of an RPG game. The user playing is able to choose their own characters, load in characters, and save their creation at any point in the battle via file I/O.

Built with:

This game is built with C++ and executes in the console.

Tic-Tac-Toe

Inspiration:

Tic-tac-toe is as simple as can be, but it's a game that stands the test of time. I was inspired to build this classic game to familiarize myself with the project planning structure of brainstorming user stories, constructing wireframes, and writing pseudocode.

What it does:

This program allows two people to play each other in a game of tic-tac-toe. It keeps track of whose turn it is, provides a warning if one of the players attempt to mark a space that is already occupied, and conveys the results when the game ends.

Built with:

This game is built with HTML, CSS, and JavaScript.
